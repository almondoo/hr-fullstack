# 09. MVP リリース DoD（Definition of Done）チェックリスト

> **目的**: 本番リリース可否を判断するための確認項目一覧
> **最終更新**: 2026-06-03
> **参照**: `docs/06_remaining_tasks.md` T6 / `docs/05_implementation_progress.md` §5 P4

---

## 使い方

リリース前に全項目を確認し、チェックボックスをマークする。
**未チェック項目があれば、その理由を明記したうえでリリース責任者の承認を得ること。**

---

## A. コード品質 / ビルド

- [ ] `go build ./...` が exit 0（コンパイルエラーなし）
- [ ] `go vet ./...` が exit 0
- [ ] `golangci-lint run ./...` の指摘が 0（または許容済み suppression があること）
- [ ] `pnpm run build`（フロントエンド）が exit 0
- [ ] `pnpm run typecheck`（フロントエンド）が exit 0

---

## B. テスト

- [ ] バックエンド全テスト `go test ./... -race -p 1` が exit 0
- [ ] フロントエンドテスト（`pnpm run test`）が exit 0（テストが存在する場合）
- [ ] RLS テナント越境テストが全件 PASS（`tenantdb.WithinTenant` の分離検証）
- [ ] 認証・CSRF テストが全件 PASS
- [ ] 機微PII 列暗号化テストが全件 PASS（マイナンバー・口座番号等の復号値非漏洩確認）

---

## C. セキュリティ

- [ ] `govulncheck ./...` の結果を確認し、CRITICAL / HIGH の未修正 CVE が存在しないこと（許容済みのものは理由を記録）
- [ ] `pnpm audit --audit-level=high` の結果を確認し、HIGH 以上の未修正脆弱性が存在しないこと
- [ ] GitHub Secret scanning + Push Protection が有効化済み
- [ ] `.env`、APIトークン、本番接続情報がコミット履歴に含まれていないこと（`git log` で確認）
- [ ] `SECURITY.md` の脆弱性開示ポリシー（連絡先・SLA）が実際の連絡先に更新済み
- [ ] 本番デプロイ時に `SESSION_COOKIE_SECURE=true` / `CSRF_SECURE=true` が設定されること
- [ ] `DB_SSLMODE=verify-full`（または `verify-ca`）が設定され、接続が暗号化されること
- [ ] `FIELD_ENCRYPTION_KEY` が Secret Manager からインジェクトされ、リポジトリに存在しないこと

---

## D. 本番 Dockerfile / コンテナ

- [ ] `backend/Dockerfile`（本番用マルチステージ）でイメージがビルドできること
- [ ] コンテナが非 root ユーザー（uid=65532 / `nonroot`）で実行されること
- [ ] `/healthz` エンドポイントが 200 を返すこと
- [ ] `/readyz` エンドポイントが DB 接続後に 200 を返すこと
- [ ] イメージに不要なシェル・ツール・パッケージが含まれていないこと（distroless 確認）

---

## E. 環境変数 / 設定

- [ ] `.env.example` が全環境変数を網羅しており、値がプレースホルダのみであること
- [ ] `docs/07_env_reference.md` が最新の設定と一致していること
- [ ] 本番チェックリスト（`docs/07_env_reference.md` §本番チェックリスト）が全件チェック済み
- [ ] `MIGRATE_ON_STARTUP=false` が本番設定で確認済み
- [ ] `GIN_MODE=release` が本番設定で確認済み
- [ ] `CORS_ALLOW_ORIGINS` に本番ドメインが明示設定済み

---

## F. データベース / マイグレーション

- [ ] マイグレーション `00001` 〜 `00027` が本番 DB に全件適用済み（`goose status` で確認）
- [ ] `hr_app` ロールが `NOBYPASSRLS` で作成済み（`\du hr_app` で確認）
- [ ] RLS ポリシーが全対象テーブルで `ENABLED` + `FORCED` になっていること
- [ ] マイグレーション前の DB スナップショット（バックアップ）が取得済み

---

## G. 運用ドキュメント

- [ ] `README.md` のセットアップ手順が最新化されていること
- [ ] `docs/08_migration_runbook.md` の手順でマイグレーションが実行できることを確認
- [ ] 本番インシデント対応フロー（障害発生時の連絡先・ロールバック手順）が定義済み
- [ ] `SECURITY.md` の脆弱性報告連絡先が設定済み

---

## H. コンプライアンス / 法令（要専門家確認）

> 以下は専門家（社労士・弁護士）との確認が前提です。断定的な法的助言ではなく、確認すべき観点の列挙です。

- [ ] マイナンバー取得・利用・提供・廃棄フローが番号法の特定個人情報の適正な取扱いに関するガイドラインに沿っていることを専門家と確認済み
- [ ] 健康診断結果・病歴等の要配慮個人情報の取扱いが個人情報保護法の要件を満たしていることを専門家と確認済み
- [ ] 利用規約・プライバシーポリシー（別途準備）が整備済み
