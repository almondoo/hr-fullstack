## 0. このテンプレートの使い方

1. プレースホルダ `{{ ... }}` をすべて実値に置換する（実 PII・秘密は置換時もコミットしない運用に注意）。
2. 後述「§A 取得項目の対応づけ（実装由来）」を、最新のスキーマ（`backend/db/migrations/`）と突き合わせて更新する。
3. 弁護士・社労士レビュー → 指摘反映 → 確定版へ。確定後はステータス行を更新する。
4. 顧客（テナント）が自社の従業員・応募者向けに掲示する版と、当社が当社サービス利用者（顧客の管理者）向けに掲示する版は **別物**。本テンプレートは主に **当社サービス（HR SaaS）利用者向け**を想定。顧客が自社の被用者に示すポリシーは顧客自身が用意する（→ DPA / `dpa.md` の役割整理を参照）。

## §A 取得項目の対応づけ（実装由来・要メンテナンス）

> **目的**: ポリシー文面が実装と乖離しないよう、取得項目をスキーマ起点で棚卸しする。
> 列の正確な定義は各 `backend/db/migrations/000NN_*.sql` を一次情報とする。専門家レビュー時の突合資料。

| 区分 | 主な格納先（ドメイン / migration） | PII 機微度 | 備考 |
|---|---|---|---|
| アカウント / 認証 | `auth`（00002/00003） | 中 | パスワードは argon2id 等でハッシュ保存（平文・可逆保存禁止） |
| 従業員・組織・契約 | `employee` / `department`（00004） | 中 | 氏名・所属・雇用契約 |
| 勤怠 | `attendance`（00005, 00032） | 中 | 36協定・割増の計算対象。法令値は設定/migration 由来 |
| 休暇（年休等） | `leave`（00007） | 中 | 比例付与・5日取得義務の対象 |
| **マイナンバー** | `mynumber`（00010） | **要配慮 / 番号法** | 列暗号 + 専用復号権限 + 利用提供ログ。平文非格納 |
| 社会保険・電子申請 | `govfiling`（00014, 00032） | 高 | 届出 payload に機微情報を残さない設計 |
| 法定帳簿 | `ledger`（00015） | 中〜高 | 法定三帳簿・保存年限 |
| 就業規則・労使協定 | `workrule`（00019） | 低〜中 | 36協定書面の不透明 ID 参照 |
| 採用（応募者） | `applicant` / `selection` / `offer` / `interview` / `hiring`（00018, 00020, 00021, 00024, 00025） | 中 | 応募者 PII |
| 評価・目標・1on1・タレント | `evaluation` / `goal` / `oneonone` / `talent`（00022, 00012, 00023, 00026） | 中 | 人事評価情報 |
| 年末調整 | `yearend`（00031） | 高 | 税・控除情報 |
| 通知 | `notification`（00009, 00028） | 低 | 監査・ログに PII を残さない |
| 監査ログ | `platform/audit` | 低（PII 非格納設計） | ハッシュチェーン。`resource_id` に機微値を入れない |

> ⚠️ 上表は雛形であり、最新のマイグレーション一覧（現状 `00001`〜`00032`）と必ず突合すること。
> 列レベルの取得項目・利用目的・保存年限は、確定前に弁護士・社労士の確認を受ける。
