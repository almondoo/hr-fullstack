# 01 / HR SaaS 要件カタログ（ドメイン知識）

Phase 1 で **抜け漏れを防ぐチェックリスト** として使用する。
各要件には **ID** と **優先度ヒント（M=Must / S=Should / C=Could）** を付与（MVP判断用の目安）。
ID接頭辞は領域コード：`CORE`(基盤) `LM`(労務) `ATS`(採用) `TM`(タレマネ) `NFR`(非機能) `INT`(連携) `CMP`(法令)。

> 注意：法令系（CMP / LM の制度対応）は **2025–2026年時点の一般情報** に基づく要約。
> 実装時は社労士・弁護士と官公庁一次情報で必ず確認すること。

## H. 競合プロダクト参照（機能網羅の当て先）

要件の妥当性チェックに、国内主要プロダクトの機能セットを当て先として使う（模倣でなく網羅性確認の目的）：

- **人事労務**：SmartHR、freee人事労務、ジョブカン労務、マネーフォワード クラウド人事労務、KING OF TIME（勤怠）
- **ATS**：HRMOS採用、sonar ATS、i-web、リクナビHRTech
- **タレマネ**：カオナビ、HRBrain、タレントパレット、One人事

> これらは「標準機能の相場観」を掴むための参照。**UIや独自仕様の複製はしない**。
